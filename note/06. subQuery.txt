 ① Sub Query : 메인 QUERY(SQL문) 안에 내포된 QUERY
 ② 비교 연산자의 오른쪽에 기술, 반드시 괄호로 둘러쌓여야 함 
 ③ 서브 쿼리는 메인 쿼리가 실행되기 이전에 한번만 실행이 됨
 ④ 서브쿼리 종류(1) 단일행 서브쿼리: 서브쿼리 결과가 단일행
 ⑤ └ 단일행일 때 사용가능한 연산자: =, >, <, >=, <=, !=
 ⑥ 서브쿼리 종류(2) 다중행 서브쿼리: 서브쿼리 결과가 2행 이상
 ⑦ └ 다중행일 때 사용가능한 연산자: IN, ALL, ANY(SOME), EXISTS
 ⑧ ALL: 메인 쿼리의 비교 조건이 서브쿼리 검색 결과와 모두 만족하면 참
 ⑨ ANY, SOME : 메인 쿼리의 비교 조건이 서브 쿼리의 검색 결과와 하나라도 만족하면 참
 ⑩ IN : 메인 쿼리의 비교 조건(‘=’ 연산자로 비교할 경우)이 서브 쿼리의 결과 중에서 하나라도 일치하면 참
 ⑪ EXISTS : 메인 쿼리의 비교 조건이 서브 쿼리의 결과 중에서 만족하는 값이 하나라도 존재하면 참
