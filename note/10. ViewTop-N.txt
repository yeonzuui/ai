 ① VIEW: 가상의 테이블, 이미 존재하고 있는 테이블에 제한적으로 접근하도록 하기 위함
 ② 뷰 생성(이미 있으면 교체)
	CREATE OR REPLACE 뷰이름 AS SELECT 필드 FROM 테이블명 WHERE 조건;
 ③ (1) 단순뷰: 하나의 테이블을 이용해서 만든 뷰(가상의 테이블은 물리공간과 데이터가 따로 없음)
 ④ SELECT * FROM TAB; 가상테이블 포함 SELECT * FROM USER_TABLES; 가상테이블 미포함
 ⑤ 기본 테이블의 데이터가 변경되면 뷰에도 반영
 ⑥ 단순VIEW 생성시 NOT NULL 필드를 미포함한 경우 INSERT 불가능
 ⑦ VIEW 제한 조건 
	1. WITH CHECK OPTION 추가: 뷰의 조건에 해당되는 데이터만 INSERT, MODIFY, DELETE 가능
 	2. WITH READ ONLY 추가: 읽기 전용 뷰 생성
 ⑧ (2) 복합뷰: 2개 이상의 테이블로 구성한 뷰, 가상의 필드를 이용한 뷰(컬럼에 별칭을 사용)
 ⑨ 복합뷰는 DML문을 제한적으로만 이용(ONLY SELECT)
 ⑩ (3) INLINE-VIEW: FROM절에 오는 서브쿼리는 INLINE-VIEW로 작용
 ⑪ TOP-N 구문(TOP 1-10등, 11-20등...): 순위 매기기 위함
 ⑫ ROWNUM: 테이블로부터 가져온 출력될 데이터(행)의 논리적인 순서. 
 ⑬ -- 1단계: INLINE VIEW - RANK 기준으로 정렬(ORDER BY)
 ⑭ -- 2단계: INLINE VIEW- ROWNUM(RANK 포함), ALIES(RN) 필수
 ⑮ -- 3단계: 메인함수(ROWNUM 포함): WHERE에 RN에 대한 조건 작성
