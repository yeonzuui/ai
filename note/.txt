 ① INDEX: 조회를 빠르게, SQL 명령문(SELECT)의 처리 속도를 향상
 ② 인덱스의 내부 구조는 b*TREE 형식으로 구성: 아스키코드 순 가지치기
 ③ 인덱스 생성:  CREATE INDEX 인덱스명 ON 테이블(필드)
 ④ 인덱스명: IDX_테이블명_찾을필드명
 ⑤ 인덱스 단점 1. 인덱스(b*TREE)를 위한 추가적인 공간 필요
	        2. 인덱스를 생성하는데 시간 소요
 ⑥ 데이터의 변경 작업(INSERT/UPDATE/DELETE)이 자주 일어날 경우에는 오히려 성능 저하(느려짐)
 ⑦ 인덱스 사용해야 하는 경우:
	- 테이블에 행의 수가 많을 때
	- 검색 결과가 전체 데이터의 2%~4% 정도 일 때 
	- WHERE 문에 해당 컬럼이 많이 사용될 때
	- JOIN에 자주 사용되는 컬럼이나 NULL을 포함하는 컬럼이 많은 경우
 ⑧ 인덱스를 사용하지 말아야 하는 경우: 위의 경우에 반대되는 경우
